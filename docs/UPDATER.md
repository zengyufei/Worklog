# Tauri v2 Updater Configuration Guide

This guide focuses only on configuring the updater (no CI, no frontend), using `tauri.conf.json` for a Tauri v2 app.

---

## 1. Prerequisites: generate signing keys

Tauri’s updater **requires signed builds**, so you must generate a keypair first. [web:23][web:11]

```bash
tauri signer generate
```

This command creates:

- A **private key** (used to sign installers in CI / when building).
- A **public key** (string content) that you will paste into the config as `pubkey`. [web:23]

Keep the private key safe; losing it blocks future updates for existing installs. [web:23]

---

## 2. Enable updater artifacts in `tauri.conf`

In your `src-tauri/tauri.conf.json` (or `.json5` / `.toml` equivalent), enable updater artifacts under `bundle`. [web:1][web:18][web:21][web:23][web:26]

```jsonc
{
  "tauri": {
    "bundle": {
      "createUpdaterArtifacts": true
    }
  }
}
```

- `createUpdaterArtifacts: true` tells the bundler to generate `.sig` files and other assets required for updates. [web:1][web:18][web:11]
- If you are migrating from Tauri v1, you can temporarily set `"v1Compatible"` instead of `true`, but new apps should use `true`. [web:1][web:18][web:21]

---

## 3. Basic updater plugin configuration

Next, configure the updater plugin block with your **public key** and **endpoints**. [web:1][web:18][web:21][web:23][web:26]

```jsonc
{
  "tauri": {
    "bundle": {
      "createUpdaterArtifacts": true
    },
    "plugins": {
      "updater": {
        "pubkey": "PASTE_YOUR_PUBLIC_KEY_STRING_HERE",
        "endpoints": [
          "https://releases.myapp.com/{{target}}/{{arch}}/{{current_version}}",
          "https://github.com/your-user/your-repo/releases/latest/download/latest.json"
        ]
      }
    }
  }
}
```

Key fields: [web:1][web:18][web:21][web:23][web:26]

- `pubkey`  
  - Must be the **public key content** generated by `tauri signer generate`.  
  - It **cannot** be a file path; paste the actual text string.  

- `endpoints`  
  - An array of URLs that the app will query to determine if an update is available.  
  - In production, Tauri enforces **HTTPS** by default.  
  - If a URL returns a **non-2xx** status, Tauri automatically tries the next URL.  

You can mix a dynamic server endpoint (using `{{target}}`, `{{arch}}`, `{{current_version}}`) and a static JSON (like `latest.json` hosted on GitHub Releases). [web:1][web:18][web:21][web:23][web:26]

---

## 4. Optional: allow HTTP for local testing

When testing locally you may want to serve `latest.json` over plain HTTP (e.g. `http://localhost:3000/latest.json`). For that, add `dangerousInsecureTransportProtocol: true`. [web:1][web:18][web:11]

```jsonc
{
  "tauri": {
    "bundle": {
      "createUpdaterArtifacts": true
    },
    "plugins": {
      "updater": {
        "pubkey": "PASTE_YOUR_PUBLIC_KEY_STRING_HERE",
        "endpoints": [
          "http://localhost:3000/latest.json"
        ],
        "dangerousInsecureTransportProtocol": true
      }
    }
  }
}
```

- This flag allows **non-HTTPS** endpoints and should only be used in development or controlled environments. [web:1][web:18][web:11]

---

## 5. Windows-specific install behavior (optional)

You can tweak the Windows installer behavior via the `windows` subkey. Example: passive install mode. [web:11]

```jsonc
{
  "tauri": {
    "bundle": {
      "createUpdaterArtifacts": true
    },
    "plugins": {
      "updater": {
        "pubkey": "PASTE_YOUR_PUBLIC_KEY_STRING_HERE",
        "endpoints": [
          "http://localhost:3000/latest.json"
        ],
        "dangerousInsecureTransportProtocol": true,
        "windows": {
          "installMode": "passive"
        }
      }
    }
  }
}
```

This controls how the update installer behaves on Windows (e.g., whether it shows UI). [web:11]

---

## 6. Summary of important configuration keys

| Key                                   | Where                               | Purpose                                                                                   |
|---------------------------------------|--------------------------------------|-------------------------------------------------------------------------------------------|
| `createUpdaterArtifacts`              | `tauri.bundle`                       | Enables generation of signatures and updater artifacts during build. [web:1][web:11]      |
| `pubkey`                              | `tauri.plugins.updater`              | Public key used to verify signed update artifacts. [web:1][web:18][web:23]                |
| `endpoints`                           | `tauri.plugins.updater`              | One or more URLs providing update metadata / assets. [web:1][web:18][web:23][web:26]      |
| `dangerousInsecureTransportProtocol`  | `tauri.plugins.updater`              | Allows HTTP (non-HTTPS) endpoints, for dev only. [web:1][web:18][web:11]                  |
| `windows.installMode`                 | `tauri.plugins.updater.windows`      | Controls Windows installer behavior during update. [web:11]                               |
